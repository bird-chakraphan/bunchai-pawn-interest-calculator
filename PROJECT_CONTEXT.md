# PROJECT_CONTEXT: Pawn-Interest-Calculator

## What This Project Is

Pawn-Interest-Calculator is the local Codex project folder for `PawnInterestCalculator`, a compact Framer React + TypeScript component for calculating Thai pawnshop interest (`ดอกเบี้ยจำนำ`).

The main user-facing component is Thai-language and intended to be embedded in Framer. It calculates automatically after the user enters:

- `วันเริ่ม / ต่อดอกล่าสุด`
- `ยอดจำนำ`
- `โปรโมชัน`
- `รายการ`, currently `ต่อดอก` or `ไถ่ของ`

The folder also contains a spreadsheet-generation script and generated review artifacts for checking the interest formula.

## What Has Already Been Built

- Built the compact Thai pawnshop interest calculator component for Framer.
- Calculation auto-updates after required data is entered; there is no manual calculate button.
- Supports start/latest renewal date, pawn amount, promotion selection, and transaction selection.
- Promotion UI uses segmented buttons instead of a dropdown.
- `โปรแสน (1.5%)` displays as `โปรแสน`, while internal calculation still uses `1.5%`.
- Native OS date input was replaced with a custom Thai date picker.
- Date picker flow is:
  1. select year
  2. select month
  3. select date
- Date display uses Buddhist year, for example `6 มี.ค. 2569`.
- Future dates are disabled in the picker and clamped in date updates.
- Input date and today text both use Thai short-date format.
- `ไถ่ของ` result shows the large main amount as `loanAmount + calculation.interestAmount`.
- `ไถ่ของ` result includes separate rows for:
  - `เงินต้น`
  - `ดอกเบี้ย`
- `calculation.interestAmount` rounds up with `Math.ceil()` and displays as an integer with no decimals.
- Expandable `ดูวิธีคิด` was removed.
- Header text was removed from the rendered UI:
  - `คำนวณดอกเบี้ยจำนำ`
  - `คำนวณอัตโนมัติเมื่อกรอกข้อมูลครบ`
- The outer `.pawn-calculator` is transparent and has no border, stroke, shadow, padding, or border radius.
- The spreadsheet is a formal source of truth alongside the Framer component:
  - interest rounding in the sheet should mirror component `Math.ceil()` behavior
  - calculation examples should be based on the component logic
  - changes to core calculation rules should update both `src/PawnInterestCalculator.tsx` and `scripts/build_pawn_interest_sheet.mjs`

## Important Decisions

- Internal date storage remains Gregorian `YYYY-MM-DD`.
- Display dates use Thai Buddhist year.
- Keep the calculation logic anniversary-based.
- Do not divide days by 30 to calculate months.
- Month boundaries use calendar anniversaries with end-of-month fallback.
- Keep desktop side-by-side layout unchanged unless the user explicitly asks otherwise.
- Do not return to a native browser/OS date picker, because customer-facing consistency matters.
- Segmented buttons should use explicit flex centering for Thai text alignment.
- Current responsive layout behavior is primarily pure CSS `flex-wrap`.

## Known Issues

- When the layout stacks on smaller widths, `.pawn-result-wrap` may not fill the full container width.
- This happens because stacking is caused naturally by `flex-wrap`, not by an explicit breakpoint.
- CSS cannot reliably detect "this item has wrapped" in this implementation.
- A previous `@container (max-width: 639px)` attempt caused layout issues and was removed.
- Some old controls/CSS may still exist even if no longer rendered, including:
  - `titleLabel`
